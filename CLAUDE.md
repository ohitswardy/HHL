# HardhatLedger — Claude Code Instructions

## Project Overview
HardhatLedger is a unified business platform for a construction materials supplier. It replaces Loyverse POS + QuickBooks with a single integrated system covering Inventory, POS, and Accounting.

## Repository Layout
```
HHL/
├── hardhatledger-api/      # Laravel 12 backend (PHP 8.2)
├── hardhatledger-web/      # React 19 + TypeScript frontend
└── HardhatLedger_MasterPlan (1).docx
```

## Tech Stack

### Backend (hardhatledger-api)
- Laravel 12, PHP 8.2
- MySQL 8.0 via XAMPP (database: `hardhatledger`)
- Laravel Sanctum (API token auth)
- Spatie Laravel Permission (RBAC)
- Barryvdh DomPDF (PDF receipts/reports)
- API versioned at `/api/v1`

### Frontend (hardhatledger-web)
- React 19 + TypeScript 5.9, Vite 8
- Tailwind CSS 4, Zustand, React Query (TanStack), React Router 7
- React Hook Form + Zod, TanStack Table, Recharts
- Dev server: `http://localhost:5173`
- API base URL: `http://localhost:8000/api/v1`

## Key Conventions

### Backend
- All tables include `branch_id` for future multi-branch support
- Soft deletes (`deleted_at`) on all major entities
- Business logic lives in Services, not Controllers
- Controllers use Form Request classes for validation
- API responses use Resource transformers
- Double-entry accounting is enforced via `JournalService`
- Transaction numbers are generated by `TransactionNumberService`

### Frontend
- Auth token stored in `localStorage` under key `hhl_token`
- User object stored under `hhl_user`
- Axios instance is at `src/lib/api.ts` — add all API calls through this
- Zustand stores: `authStore` (auth + RBAC), `cartStore` (POS cart)
- Feature modules are under `src/modules/<module>/pages/`
- Shared UI primitives are in `src/components/ui/`

## RBAC Roles
- `Sales Clerk` — POS operations only
- `Admin` — Full access except user management
- `Manager` — Reporting and approvals
- `Super Admin` — All access including user management

## Design System
- Primary: Navy `#1B3A5C`
- Accent: Amber `#F5A623`
- Style: Neo-Skeuomorphic cards, POS speed-first
- Use Tailwind utilities; avoid inline styles unless dynamic

## Dev Environment
- XAMPP must be running (Apache + MySQL)
- Backend: `cd hardhatledger-api && php artisan serve`
- Frontend: `cd hardhatledger-web && npm run dev`
- CORS allows `http://localhost:5173` and `http://127.0.0.1:5173`

## What NOT to Do
- Do not skip `branch_id` when creating new tables or seeding data
- Do not bypass Sanctum middleware on protected routes
- Do not write business logic directly in Controllers — use Services
- Do not hard-code prices; always resolve via `PricingService`
- Do not use `git add -A` blindly — check for `.env` files first
